## Product Thinking

### Core Job-to-be-Done
One sentence: what is the user trying to accomplish when they arrive?
What does success look like *for them*?

### User Mental Model
What does the user already believe/expect on arrival?
What did the design need to match, challenge, or correct?

### What I Chose NOT to Build  *(often your strongest signal)*
What was on the table and deliberately cut, simplified, or deferred — and why?
What would have gone wrong if you'd shipped the "obvious" bigger version?

### Information Architecture
What information does the user need, in what order?
How did you decide what's first / above the fold / behind a tap?
Where does complexity live, and how is it contained?

### User Flows  *(usually more than one)*
For each path: entry → key decisions → outcome.
What determines which path a user takes? Where do they drop off, and how does the design catch them?
